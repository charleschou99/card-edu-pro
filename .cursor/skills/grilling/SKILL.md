---
name: grilling
description: Interview the user relentlessly about a plan or design. Use when the user wants to stress-test a plan before building, or uses any 'grill' trigger phrases.
---

Interview me relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one-by-one. For each question, provide your recommended answer.

Ask the questions one at a time, waiting for feedback on each question before continuing. Asking multiple questions at once is bewildering.

If a question can be answered by exploring the codebase, explore the codebase instead.

## Ponytail coexistence

While this skill is active, **suspend ponytail**: do not apply the lazy-senior ladder, do not minimize scope, skip features, or write code when the job is to interview and decide.

When shared understanding is reached and the user is ready to **implement**, close with:

> Grill complete. For implementation: **ponytail full** — follow the ladder, minimal diff. Attach the ponytail skill or say "ponytail full — implement what we decided" if the agent drifts back to planning or over-scaffolding.
